1) выжимка из условия: 
есть 2 мерная сетка-карта, изначальный размер и границы не заданы.
есть  точка - источник песка~камушков
есть набор строк формата
x1,y1 -> x2,y2 -> .... -> xn,yn
например 12, 13 -> 13, 18 -> 20, 18
т.е. последовательность точек.
каждая такая последовательность "рисует" на карте препятствие в виде ломаной линии
после этого нужно провести симуляцию падения камушков из источника. 
автомат клеток с песком:
стартует из источника
упасть вниз
иначе упасть вниз вправо
иначе упасть вниз влево
иначе остановиться
нужно найти количество камушков, которые остановятся. 
не остановленые считаю такие камушки, которые доходят до нижней границы всех препятствий - как только один камушек доберется до такой границы - все прочие пойдут по его дорожке.

"используй map<vertical int, map<horizontal int, char>> !"

2) решение:
	завожу карту, задаю ей смещение от точкой источника. 
	считываю строки - считываю первую точку
	расширяю карту, чтобы она содержала эту точку
	запоминаю предыдущую точку, считываю следующую точку: если нет - перехожу к следующей строке; если есть - расширяю карту, чтобы она содержала эту точку,  прохожу по всем точкам от предыдущей до текущей, помечаю все как препятствия.
---- есть карта .-свободные места, * - источник,  #-препятствие. o- остановившиеся камушки, ~- "осыпающийся" камушек.
	вывожу карту, для визуального контроля и порадоваться промежуточному результату		
	произвожу симуляцию камушков.
	это задача поиск пути  в глубину с условием движения камушков. нужно посчитать количество "тупиковых вариантов"
	добавляю в стек источник
	если нижняя/нижнеправая/нижнелевая свободна - добавляю в стек, спускаюсь по стеку.
	если ни одна не свободна - инкрементировать, взять следующую из стека.
	условие выхода - добрались до нижней границы карты.
